## Description

When selecting a Doki theme by its name (e.g., themes containing "Doki" in the name), the theme is not properly applied. The selection appears to go through, but the theme does not visually change in the editor.

## Steps to Reproduce

1. Open the extension's theme picker
2. Search for or select a Doki theme by name
3. The theme is selected but not visually applied

## Expected Behavior

The selected Doki theme should be applied correctly, just like any other theme.

## Actual Behavior

The theme selection registers, but the editor does not reflect the selected Doki theme.

## Environment

- Extension version: latest
- VS Code version: latest
- OS: Windows

## Additional Context

This seems specific to themes that include "Doki" in their name. Other themes work as expected.
